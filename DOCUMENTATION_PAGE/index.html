üîπ Files Structures

All the file are well organized, its so easy to work with the template. Below a full description of all the folders and files.
First Unzip the portFolioReact.zip file. You will find 3 folder (css, public, src) and 7 other files.


Css folder: In this directory, the common css rules for of the template (index.min.css and index.min.css.map).
These 2 files are generated by modifying the sass file index.scss. You don't have to editing this 2 files, you only, if needed, have to change the index.scss file in the /src/assets/css directory

Public folder: In this directory, there are two sub-folders:

  • portfolio_images: It containes the images of the portfolio section. You have to put in this folder the images of your project.(See the Porftolio section of this instructions to see the ideal sizes of the images)
  • news_images: It containes the images of the news section. You have to put in this folder the images of news section (See the News section of this instructions to see the ideal sizes of the images)

Src folder:
I this folder ther are the 2 main React files (main.jsx and App.jsx) and 3 sub-folders:
  • assets:
    /css:The main css files of the template. If you modifiy it, it will automatically update the index.min.css and index.min.css.map files in the css directory.
    /news: it containes the news.json file that containes the articles of the news section
    /works: it containes the portfolio.json file that containes the articles of the portfolio section
  • Components:
    This folder contains all the components of the website. All components are React Functional components. They can be imported and reused in different pages.
    /common: The 3 common-components of the react Porftolio (Footer, Navbar, SwitchDark)
    /images: The folder of the images used in this website
    /style: The css files of each component in order to simplify the customization (each component has its own css file)
  • Redux:
    This folder contains two redux files and functions used, simple speaking by the component SwitchDark to switch from the ligth to dark theme.

Other Files: The rest of the files are .gitignore, index.html, package-lock.json, package.json, readme and vite.config.js file.

üîπ Environment Setup & Local Installation

(Node packages and Vite React)

In order to use or run ReactFolio on your system, Firstly, you have to setup the React environment in the system, Follow the following steps to make your system compatible for using ReactFolio or any other React project.

1. Install Node.js
Node.js is a JavaScript run time built needed to run a React App. You can install the recommended version of Node.js from the official website https://nodejs.org/en/

2. Install required NPM Packages

This app requires the following packages:

  • bootstrap
  • react-bootstrap
  • react-redux
  • @reduxjs/toolkit
  • @emailjs/browser
  • sass
  • react-router-dom

To install them, first go inside the main folder, open a terminal and digit following:

npm install react-bootstrap bootstrap react-redux @reduxjs/toolkit @emailjs/browser sass react-router-dom

After finish the installation, you should see, in the packages.json file the following dependencies:

2. Install React locally using Vite
Vite https://vitejs.dev/guide/ A development environment used to develop applications with many frameworks inside like, in this case, React.js. To install it locally on your computer you can:

  • Open a terminal, navigate to the main directory you and, once inside, digit npm create vite@latest . üëâ Please note and do not forget the dot . üëà at the end of the command. It will install Vite directly in the folder.
  • From the options, please choose React and Javascript
  • Once finish installation, digit npm install
  • Then digit npm run dev

Typically the app runs on the following path:

hhtp://localhost:5173/

In your browser (or click on the link) go the path above and you will see the initial page of a new React application.

Now copy from the template folder the following files and folders:

  • index.html
  • css folder
  • public folder
  • src folder

    • Go to the main folder (the one where you installed Vite and all the npm packages) and paste the files and folders choosing "yes" when asked to overwrite present files

üîπ Portfolio section

This section shows how to customize the portfolio, how to add and remove projects editing the works.json file

PORTFOLIO

Component: Portfolio.jsx

Directory: /src/Components/

In the /assets folder is present a JSON file called portfolio.json

In this file is present a list of Demo Projects, that you can actually see in the demo. Opening this file with a Code Editor like, for example Visual Studio, you can change the informations of the projects.

Parameters of the JSON file:

thumbImage:
First of all, put inside this directory

/public/portfolio_images

the image you want to use for the project

The ideal size of the image is: 1000px x 500px

Then change the name of the images in the "thumbImage" property of the Json object

Title:
Simply insert the project title

Category:
Insert the description of the project. Important, each new category will generate a filter key at the top of the portfolio, to filter the projects by its category. Just for information, this feature is controlled by the script present between lines 7 and 55 of the Portfolio.jsx component

External Link:
You can put the path to the projects in case it stands on another domain. For information, this link will open in a new tab of the browser, link is set to target ="_blank" into the Single_post.jsx page

Description:
You can insert the full description of the projects.

‚öôÔ∏è Number of Columns:
You can change the number of column (now set to 3 => row-cols-md-3) by simply changing the Portfolio.jsx file as follows:

row row-cols-1 row-cols-sm-2 row-cols-md-3 g-3

For example to

row row-cols-1 row-cols-sm-2 row-cols-md-4 g-3

To get the 4 column layout and so on.

This component is composed using Bootsrap5, so you can easily customize everything following the Bootsrap5 logics

‚ö° NOTE

The most valuable feature of this portfolio component is that automatically generates the category filter buttons, as long as you enter categories into the portfolio.json file. See an example of this portfolio fetching the datas from a food market type json

Deployment Live

There are a lot of deployment platform, like Vercel, Render, Heroku and many more.
I used many times Vercel and found it really smooth and quite easy to understand.
You can follow the below instructions or, if you prefer, follow the official Vercel Guides on the official Vercel website

1. GITHUB
If you have a github account login or if not register in github.
Create a repository and connect to ReactFolio via git remote add method.
After adding the remote url push ReactFolio to that git repository.

2. VERCEL
If you don't have a Vercel account, create a new one simply signin up using your github account.
Once you connected, go the Vercel DashBoard Click add new project. Select project from the GitHub repository where you stored ReactFolio

After importing the repository you will be redirected to deployment page here you can change the project name and many other. After selecting a name for the project just click on deploy button. It will take some time and after it will be live and a url will be given which can modified later on.
Vercel will automatically detect the framework and any necessary build settings.

To view your deployment, select the Project in the dashboard and then select the Domain. This page is now visible to anyone who has the URL.

Every time you will make any change in your file, after push to GitHub, Vercel will automatically change the deployment and you will immediately see the changes in your LIVE application.
